# MCP Server: поговоріть зі своєю моделлю

Команда `raps mcp` запускає сервер Model Context Protocol (MCP), що дозволяє ШІ-помічникам взаємодіяти з Autodesk Platform Services природною мовою.

![Демонстрація MCP-сервера RAPS](/terminal/mcp-demo.gif)

> **Нове у v3.0.0** — MCP-сервер є основним нововведенням.
> **Розширено у v4.1.0** — Тепер з масовими операціями адміністрування, Issues, RFI та розширеними функціями ACC.
> **Покращено у v4.1.1** — Сервер запускається без облікових даних та надає інтерактивні інструкції з налаштування.
> **Вдосконалено у v4.4.0** — Тепер з 51 інструментом, включаючи дозволи папок, життєвий цикл операцій та повний CRUD для модулів ACC.
> **Розширено у v4.5.0** — Тепер із 72 інструментами: спеціальні API-запити, портфельні звіти, шаблони, вебхуки, Design Automation, Reality Capture та інше.

## Від "чату з PDF" до "чату з інструментами"

Ми виходимо за межі ери "чату з PDF". Наступний рубіж — це **чат з інструментами**.

Якщо ви коли-небудь пробували вставити JSON-метадані на 500 МБ з моделі Revit у ChatGPT, ви знаєте цей біль: "Message too long." RAG (Retrieval-Augmented Generation) допомагає, але він статичний. Він не може запитати поточний стан моделі або виконати дію.

На допомогу приходить **Model Context Protocol (MCP)**.

## Що таке MCP?

[Model Context Protocol](https://modelcontextprotocol.io/) — це відкритий стандарт, який дозволяє ШІ-помічникам використовувати зовнішні інструменти. Замість того, щоб завантажувати дані в контекстне вікно, LLM має доступ до набору інструментів у реальному часі.

З MCP-сервером RAPS ви можете попросити свого ШІ-помічника:

- Знаходити конкретні BIM-елементи за властивостями ("всі екземпляри 'Basic Wall' площею понад 100 м²")
- Створювати та керувати бакетами динамічно
- Завантажувати файли та запускати трансляції
- Переглядати проєкти BIM 360/ACC у реальному часі
- Перевіряти статус трансляції та отримувати результати
- Запитувати метадані моделі у реальному часі без обмежень розміру
- **Масово керувати користувачами** в кількох проєктах ACC/BIM 360
- **Відстежувати та керувати Issues** у будівельних проєктах
- **Запитувати RFI, Assets, Submittals та Checklists**
- **Навігувати папками та елементами** з повним контролем дозволів

Це з'єднує розрив між жорсткими, складними структурами даних BIM та інтерфейсом природної мови ШІ.

## Швидкий старт

```bash
# Запуск MCP-сервера
raps mcp
```

Сервер працює в режимі stdio, очікуючи MCP-запити від ШІ-помічників.

**Облікові дані не потрібні для запуску.** Сервер запускається навіть без налаштованих облікових даних APS. Коли ви запитаєте ШІ-помічника про статус автентифікації, він надасть покрокові інструкції з налаштування. Це означає, що ви можете спочатку додати RAPS до свого ШІ-помічника, а потім інтерактивно пройти процес налаштування.

## Конфігурація

### Claude Desktop

Додайте до файлу конфігурації Claude:

**macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
**Windows:** `%APPDATA%\Claude\claude_desktop_config.json`
**Linux:** `~/.config/claude/claude_desktop_config.json`

**Мінімальне налаштування (з подальшою допомогою):**

```json
{
  "mcpServers": {
    "raps": {
      "command": "raps",
      "args": ["mcp"]
    }
  }
}
```

Після додавання перезапустіть Claude та запитайте "What's my auth status?" — RAPS проведе вас через налаштування облікових даних.

**Повне налаштування (з обліковими даними):**

```json
{
  "mcpServers": {
    "raps": {
      "command": "raps",
      "args": ["mcp"],
      "env": {
        "APS_CLIENT_ID": "your_client_id",
        "APS_CLIENT_SECRET": "your_client_secret"
      }
    }
  }
}
```

### Cursor IDE

Додайте до `.cursor/mcp.json` у вашому проєкті:

```json
{
  "mcpServers": {
    "raps": {
      "command": "raps",
      "args": ["mcp"]
    }
  }
}
```

> **Примітка:** Cursor використовує змінні середовища з вашої оболонки, тому переконайтеся, що `APS_CLIENT_ID` та `APS_CLIENT_SECRET` встановлені.

## Як це працює: виконання інструментів у реальному часі

Замість завантаження даних у контекстне вікно, LLM має доступ до набору інструментів. Коли ви ставите запитання, процес виглядає так:

1. **Ви:** "Знайди всі екземпляри 'Basic Wall' у файлі 'Hospital.rvt', які мають площу понад 100 м²."
2. **LLM:** *Думає... Мені потрібно перевірити метадані моделі.*
3. **LLM (дія):** Викликає `raps model:query --file Hospital.rvt --category Walls`
4. **RAPS:** Повертає відфільтрований JSON-фрагмент.
5. **LLM (відповідь):** "Я знайшов 12 стін, що відповідають вашим критеріям. Ось їхні ID..."

Вам не потрібно писати SQL або складні запити з фільтрами. Ви просто запитуєте. ШІ перекладає вашу природну мову у відповідні команди RAPS.

## Приклади діалогів

### Перше налаштування (без облікових даних)

> **Ви:** Який мій статус автентифікації?

> **ШІ:** Дозвольте перевірити ваш статус автентифікації.
>
> **Потрібне налаштування**
>
> Облікові дані APS ще не налаштовані. Ось як почати:
>
> 1. Перейдіть на https://aps.autodesk.com та створіть безкоштовний обліковий запис
> 2. Створіть новий застосунок, щоб отримати Client ID та Secret
> 3. Додайте облікові дані до конфігурації MCP:
>
> ```json
> {
>   "mcpServers": {
>     "raps": {
>       "command": "raps",
>       "args": ["mcp"],
>       "env": {
>         "APS_CLIENT_ID": "your_client_id",
>         "APS_CLIENT_SECRET": "your_client_secret"
>       }
>     }
>   }
> }
> ```
>
> Після додавання облікових даних перезапустіть ШІ-помічника для застосування змін.

### Перевірка статусу автентифікації (з обліковими даними)

> **Ви:** Який мій поточний статус автентифікації?

> **ШІ:** Дозвольте перевірити ваш статус автентифікації.
>
> Статус автентифікації:
> - 2-legged OAuth: ✓ Дійсний
> - 3-legged OAuth: Не авторизовано
>
> Доступність інструментів:
> - ✓ OSS (bucket_*, object_*) — доступні
> - ✓ Derivative (translate_*) — доступні
> - ✗ Data Management (hub_*, project_*) — потрібна 3-legged автентифікація
>
> Для доступу до даних BIM 360/ACC потрібно увійти з обліковим записом Autodesk. Виконайте `raps auth login` у вашому терміналі.

> **Ви:** Створи новий постійний бакет у регіоні US з назвою "my-project-files"

> **ШІ:** Зараз створю цей бакет для вас.
>
> ✓ Створено бакет 'my-project-files' з постійним збереженням у регіоні US.

### Перевірка бакетів

> **Ви:** До яких бакетів я маю доступ?

> **ШІ:** Дозвольте перевірити ваші доступні бакети.
>
> У вас є 3 бакети:
> - my-project-files (persistent, US)
> - temp-uploads (transient, US)

> **Ви:** Знайди всі сталеві балки в моделі лікарні, які довші за 10 метрів, та покажи їхні властивості матеріалу

## Архітектура

MCP-сервер повторно використовує всі наявні API-клієнти RAPS:

```
┌─────────────────┐     ┌──────────────────┐
│   AI Assistant  │────▶│  RAPS MCP Server │
│  (Claude, etc.) │◀────│   raps mcp       │
└─────────────────┘     └────────┬─────────┘
                                 │
     ┌───────────┬───────────┬───┴───┬───────────┬───────────┐
     │           │           │       │           │           │
┌────▼────┐ ┌────▼────┐ ┌────▼───┐ ┌─▼──┐ ┌─────▼─────┐ ┌────▼────┐
│  Auth   │ │   OSS   │ │ Deriv  │ │ DM │ │   Admin   │ │   ACC   │
│ Client  │ │ Client  │ │ Client │ │    │ │  Clients  │ │ Clients │
└────┬────┘ └────┬────┘ └────┬───┘ └──┬─┘ └─────┬─────┘ └────┬────┘
     │           │           │        │         │            │
     └───────────┴───────────┴────┬───┴─────────┴────────────┘
                                  │
                           ┌──────▼──────┐
                           │   APS APIs  │
                           └─────────────┘
```

Клієнти Admin та ACC створюються на вимогу для кожного запиту для підтримки масових операцій та API, специфічних для проєктів.

## Обмеження

- **Без завантаження файлів:** MCP наразі не підтримує передачу бінарних файлів. Використовуйте CLI для завантажень.
- **3-legged автентифікація потребує CLI:** Хоча MCP-інструменти можуть перевіряти статус автентифікації та надавати інструкції, фактичний вхід через 3-legged потрібно виконувати через `raps auth login` у терміналі з доступом до браузера.
- **Транспорт stdio:** Сервер повинен бути запущений ШІ-клієнтом, а не як демон.

## Налагодження

Увімкніть детальне журналювання:

```bash
RUST_LOG=debug raps mcp
```

Перевірте з'єднання з сервером:

```bash
echo '{"jsonrpc":"2.0","method":"initialize","params":{},"id":1}' | raps mcp
```
