# Webhook Commands

Manage webhook subscriptions to receive real-time notifications when events occur in your APS projects.

> **Note:** Webhook commands require 2-legged OAuth. Run `raps auth test` to verify credentials.

## Commands Overview

| Command | Description |
|---------|-------------|
| `raps webhook list` | List all webhooks |
| `raps webhook create` | Create a new webhook subscription |
| `raps webhook get` | Get a specific webhook |
| `raps webhook update` | Update a webhook |
| `raps webhook delete` | Delete a webhook |
| `raps webhook events` | List available webhook events |
| `raps webhook test` | Test webhook endpoint connectivity |
| `raps webhook verify-signature` | Verify webhook payload signature |

---

## raps webhook list

List all registered webhook subscriptions.

```bash
raps webhook list [--output FORMAT]
```

**Example:**

```bash
$ raps webhook list
$ raps webhook list --output json
```

---

## raps webhook create

Create a new webhook subscription for a specific event type.

```bash
raps webhook create [OPTIONS]
```

**Options:**
- `-u, --url <URL>` â€” Callback URL for webhook notifications
- `-e, --event <EVENT>` â€” Event type (e.g., `dm.version.added`)

**Example:**

```bash
$ raps webhook create -e "dm.version.added" -u "https://hooks.example.com/aps"
```

> **Note:** The callback URL must be publicly reachable. APS will send a validation request to the URL during creation.

## raps webhook get

Get details of a specific webhook.

```bash

```

**Options:**
- `-e, --event <EVENT>` â€” Event type
- `--hook-id <HOOK_ID>` â€” Hook ID
- `-s, --system <SYSTEM>` â€” System (default: `data`)

**Example:**

```bash
$ raps webhook get -e "dm.version.added" --hook-id abc123
```

## raps webhook update

Update an existing webhook subscription.

```bash

```

**Options:**
- `-e, --event <EVENT>` â€” Event type
- `--hook-id <HOOK_ID>` â€” Hook ID
- `--callback-url <URL>` â€” New callback URL
- `--status <STATUS>` â€” New status (`active` or `inactive`)
- `-s, --system <SYSTEM>` â€” System (default: `data`)

**Example:**


## raps webhook test

```bash

```

**Arguments:**
- `<URL>` â€” Webhook callback URL to test

**Options:**
- `-t, --timeout <TIMEOUT>` â€” Timeout in seconds (default: 10)

**Example:**


## raps webhook verify-signature

Verify the HMAC signature of a webhook payload to ensure authenticity.

```bash
raps webhook verify-signature --signature <SIG> --secret <SECRET> <PAYLOAD>
```

**Arguments:**
- `<PAYLOAD>` â€” The webhook payload (JSON string or `@file`)

**Options:**
- `-s, --signature <SIGNATURE>` â€” The signature from `X-Adsk-Signature` header
- `--secret <SECRET>` â€” The webhook secret

**Example:**

```bash
$ raps webhook verify-signature --signature "abc123..." --secret "my-secret" '{"event":"dm.version.added"}'
```
