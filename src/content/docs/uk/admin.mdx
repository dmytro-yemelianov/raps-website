# Команди адміністрування

Виконуйте масові операції з управління користувачами в облікових записах ACC/BIM 360. Ці команди забезпечують ефективне додавання, видалення та управління правами доступу у великих масштабах.

> **Примітка:** Команди адміністрування потребують 3-legged OAuth з правами адміністратора облікового запису. Спочатку виконайте `raps auth login`.

## Огляд команд

| Команда | Опис |
|---------|------|
| `raps admin user add` | Додати користувача до кількох проєктів |
| `raps admin user remove` | Видалити користувача з проєктів |
| `raps admin user update-role` | Оновити роль користувача в проєктах |
| `raps admin folder set-permissions` | Оновити права доступу до папок |
| `raps admin project list` | Показати список проєктів з фільтрацією |
| `raps admin operation status` | Переглянути прогрес операції |
| `raps admin operation resume` | Відновити перервану операцію |
| `raps admin operation cancel` | Скасувати активну операцію |
| `raps admin operation list` | Показати список усіх операцій |
| `raps admin user list` | Показати список користувачів облікового запису або проєкту |
| `raps admin user import` | Імпортувати користувачів до проєкту з CSV |
| `raps admin project create` | Створити новий проєкт |
| `raps admin project update` | Оновити налаштування проєкту |
| `raps admin project archive` | Архівувати проєкт |
| `raps admin company-list` | Показати список компаній облікового запису |

## Команди користувачів

### raps admin user add

Додавання користувача до кількох проєктів із зазначеною роллю.

```bash
raps admin user add <ACCOUNT_ID> <EMAIL> --role <ROLE> [OPTIONS]
```

**Аргументи:**
- `ACCOUNT_ID` - Ідентифікатор облікового запису ACC/BIM 360
- `EMAIL` - Адреса електронної пошти користувача

**Опції:**
- `--role <ROLE>` - Роль для призначення (обов'язково)
  - `account_admin`, `project_admin`, `project_manager`, `viewer`
- `--filter <REGEX>` - Фільтрувати проєкти за шаблоном назви
- `--dry-run` - Попередній перегляд змін без виконання
- `--output <FORMAT>` - Формат виводу (json, table)

**Приклад:**

```bash
$ raps admin user add "acc-123" "user@company.com" --role project_admin
Adding user to projects...
[████████████████████████████████████████] 100/100
```

### raps admin user update-role

Оновлення ролі користувача в кількох проєктах.

```bash
raps admin user update-role <ACCOUNT_ID> <EMAIL> --role <ROLE> [OPTIONS]
```

### raps admin user list

Показати список користувачів облікового запису або конкретного проєкту.

```bash
raps admin user list [OPTIONS]
```

**Опції:**
- `--account, -a` - Ідентифікатор облікового запису (за замовчуванням: APS_ACCOUNT_ID)
- `--project, -p` - Показати користувачів лише конкретного проєкту
- `--role` - Фільтрувати за назвою ролі
- `--status` - Фільтрувати за статусом (active, inactive, not_invited)
- `--search` - Пошук за email або іменем

**Приклад:**

```bash
$ raps admin user list --account "acc-123" --status active
```

### raps admin user import

Імпорт користувачів до проєкту з CSV-файлу.

```bash
raps admin user import --project <PROJECT_ID> --from-csv <FILE>
```

**Опції:**
- `--project, -p` - Ідентифікатор проєкту для імпорту (обов'язково)
- `--from-csv` - Шлях до CSV-файлу зі стовпцями: email, role_id

**Приклад:**

```bash
$ raps admin user import --project "proj-123" --from-csv users.csv
```

## Команди папок

### raps admin folder set-permissions

Оновлення прав доступу до папок для користувача в кількох проєктах.

```bash
raps admin folder set-permissions <ACCOUNT_ID> <EMAIL> --permission <LEVEL> --folder <TYPE> [OPTIONS]
```

**Аргументи:**
- `ACCOUNT_ID` - Ідентифікатор облікового запису ACC/BIM 360
- `EMAIL` - Адреса електронної пошти користувача

**Опції:**
- `--permission <LEVEL>` - Рівень доступу (обов'язково)
  - `view`, `view_download`, `upload`, `edit`, `control`
- `--folder <TYPE>` - Тип папки (обов'язково)
  - `project_files`, `plans` або URN папки для довільної папки
- `--filter <REGEX>` - Фільтрувати проєкти за шаблоном назви
- `--dry-run` - Попередній перегляд змін без виконання
- `--output <FORMAT>` - Формат виводу

**Приклад:**

```bash
$ raps admin folder set-permissions "acc-123" "contractor@partner.com" \
    --permission edit --folder project_files
```

## Команди проєктів

### raps admin project list

Показати список проєктів в обліковому записі з можливістю фільтрації.

```bash
raps admin project list <ACCOUNT_ID> [OPTIONS]
```

**Аргументи:**
- `ACCOUNT_ID` - Ідентифікатор облікового запису ACC/BIM 360

**Опції:**

- `--status <STATUS>` - Фільтрувати за статусом (`active`, `archived`, `pending`)
- `--platform <PLATFORM>` - Фільтрувати за платформою (`acc`, `bim360`)

### raps admin project create

Створення нового проєкту в обліковому записі.

```bash
raps admin project create [OPTIONS]
```

**Опції:**
- `--account, -a` - Ідентифікатор облікового запису
- `--name, -n` - Назва проєкту (обов'язково)
- `--type, -t` - Тип проєкту
- `--classification` - Класифікація проєкту (production, template, component, sample)
- `--start-date` - Дата початку (ISO 8601)
- `--end-date` - Дата закінчення (ISO 8601)
- `--timezone` - Часовий пояс (наприклад, "America/New_York")

**Приклад:**

```bash
$ raps admin project create --account "acc-123" --name "New Hospital Wing" --classification production
```

### raps admin project update

Оновлення налаштувань проєкту.

```bash
raps admin project update [OPTIONS]
```

**Опції:**
- `--account, -a` - Ідентифікатор облікового запису
- `--project, -p` - Ідентифікатор проєкту (обов'язково)
- `--name, -n` - Нова назва проєкту
- `--status` - Новий статус (active, archived, suspended)
- `--start-date` - Нова дата початку
- `--end-date` - Нова дата закінчення

**Приклад:**

```bash
$ raps admin project update --account "acc-123" --project "proj-123" --status archived
```

### raps admin project archive

Архівування проєкту (скорочення для зміни статусу на archived).

```bash
raps admin project archive --account <ACCOUNT_ID> --project <PROJECT_ID>
```

**Приклад:**

```bash
$ raps admin project archive --account "acc-123" --project "proj-123"
```

## Типові робочі процеси

### Додавання нового члена команди

```bash
# 1. Попередній перегляд проєктів, на які вплине операція
raps admin user add "$ACCOUNT_ID" "new@company.com" \
  --role project_admin --dry-run

# 2. Додати користувача до всіх проєктів
raps admin user add "$ACCOUNT_ID" "new@company.com" \
  --role project_admin

# 3. Надати права доступу до папок
raps admin folder set-permissions "$ACCOUNT_ID" "new@company.com" \
  --permission edit --folder project_files
```

### Видалення працівника, що звільняється

```bash
# 1. Видалити з усіх проєктів
raps admin user remove "$ACCOUNT_ID" "departed@company.com"

# 2. Перевірити видалення
raps admin operation status
```

### Масове оновлення ролей

```bash
# Оновити роль до project_admin для активних проєктів 2024 року
raps admin user update-role "$ACCOUNT_ID" "user@company.com" \
  --role project_admin --filter "^2024-"
```

## Коди завершення

| Код | Опис |
|-----|------|
| `0` | Усі операції виконано успішно |
| `1` | Частковий успіх (деякі елементи не вдалися) |
| `2` | Не вдалося запустити операцію |
| `3` | Користувач скасував операцію |
