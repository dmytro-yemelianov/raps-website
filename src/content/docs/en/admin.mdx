# Admin Commands

Perform bulk user management operations across ACC/BIM 360 accounts. These commands enable efficient onboarding, offboarding, and permission management at scale.

> **Note:** Admin commands require **Account Administrator** role in ACC/BIM 360. Run `raps auth login` first. Set `APS_ACCOUNT_ID` or use `--account` to specify your account.

## Commands Overview

| Command | Description |
|---------|-------------|
| `raps admin user add` | [bulk] Add user to multiple projects |
| `raps admin user remove` | [bulk] Remove user from projects |
| `raps admin user update` | [bulk] Update user role and/or company across projects |
| `raps admin user add-to-all-projects` | Add user to all active projects with an optional role |
| `raps admin user list` | List users in account or project |
| `raps admin user import` | Import users to a project from CSV |
| `raps admin folder set-permissions` | Set folder permissions across projects |
| `raps admin project list` | List projects with filtering |
| `raps admin project create` | Create a new project |
| `raps admin project create-batch` | [bulk] Create multiple projects in parallel |
| `raps admin project update` | Update project settings |
| `raps admin project archive` | Archive a project |
| `raps admin company-list` | List companies in an account |
| `raps admin operation status` | View operation progress |
| `raps admin operation resume` | Resume interrupted operation |
| `raps admin operation cancel` | Cancel in-progress operation |
| `raps admin operation list` | List all operations |

## User Commands

### raps admin user add

Add a user to multiple projects with a specified role.

```bash
raps admin user add [OPTIONS] <EMAIL>
```

**Arguments:**
- `EMAIL` - User email address to add

**Options:**
- `-a, --account <ACCOUNT>` - Account ID (defaults to current profile account)
- `-r, --role <ROLE>` - Role to assign (e.g., `"Project Admin"`, `"Document Manager"`)
- `-f, --filter <FILTER>` - Filter expression (e.g., `"name:*Hospital*,status:active"`)
- `--project-ids <FILE>` - File containing project IDs (one per line)
- `--concurrency <N>` - Parallel requests (max: 50)
- `--dry-run` - Preview changes without executing
- `--output <FORMAT>` - Output format (table, json, yaml, csv, plain)

**Example:**

```bash
$ raps admin user add "user@company.com" --account "acc-123" --role "Project Admin"
Adding user to projects...
[████████████████████████████████████████] 100/100

Results:
```


### raps admin user update

Update a user's role and/or company across multiple projects.

```bash
raps admin user update [OPTIONS] <EMAIL>
```

**Arguments:**
- `EMAIL` - User email address

**Options:**
- `-a, --account <ACCOUNT>` - Account ID
- `-r, --role <ROLE>` - New role to assign (e.g., `"Project Admin"`, `"Document Manager"`)
- `--company <COMPANY>` - Company name to assign at account level
- `--from-role <FROM_ROLE>` - Only update users who currently have this role
- `-f, --filter <FILTER>` - Filter expression
- `--project-ids <FILE>` - File containing project IDs (one per line)
- `--from-csv <FILE>` - Import updates from a CSV file (columns: email, role, company)
- `--concurrency <N>` - Parallel requests (max: 50)
- `--dry-run` - Preview changes without executing
- `--output <FORMAT>` - Output format

**Example:**

```bash
$ raps admin user update "user@company.com" --account "acc-123" --role "Document Manager"
Updating user...
[████████████████████████████████████████] 50/50

Results:

│ Status       │ Count │
├──────────────┼───────┤
│ Succeeded    │ 48    │
│ Skipped      │ 2     │
└──────────────┴───────┘
```

### raps admin user add-to-all-projects

Add a user to **all active projects** in an account with an optional role assignment. Unlike `user add`, this command targets every active project without requiring a filter — useful for onboarding new hires who need access to your entire project portfolio.

```bash
raps admin user add-to-all-projects [OPTIONS] <EMAIL>
```

**Arguments:**
- `EMAIL` - User email address to add

**Options:**
- `-a, --account <ACCOUNT>` - Account ID (defaults to `APS_ACCOUNT_ID` env var)
- `--role <ROLE>` - Role ID to assign in each project (e.g., the project administrator role ID)
- `--concurrency <N>` - Parallel requests (max: 50)
- `--dry-run` - Preview changes without executing
- `--output <FORMAT>` - Output format

**Example:**

```bash
# Add user to all active projects
$ raps admin user add-to-all-projects "new.hire@company.com" --account "acc-123"

# Add with a specific role ID
$ raps admin user add-to-all-projects "new.hire@company.com" \
    --account "acc-123" \
    --role "your-project-admin-role-id"

# Preview first with dry-run
$ raps admin user add-to-all-projects "new.hire@company.com" --dry-run
```

> **Tip:** Use `raps acc project-roles list --project <PROJECT_ID>` to look up the role IDs available in your account.

### raps admin user list

List users in an account or specific project.

```bash
raps admin user list [OPTIONS]
```

**Options:**
- `--account, -a` - Account ID (defaults to APS_ACCOUNT_ID)
- `--project, -p` - List users for a specific project only
- `--role` - Filter by role name
- `--status` - Filter by status (active, inactive, not_invited)
- `--search` - Search by email or name

**Example:**

```bash
$ raps admin user list --account "acc-123" --status active
```

### raps admin user import

Import users to a project from a CSV file.

```bash
raps admin user import --project <PROJECT_ID> --from-csv <FILE>
```

**Options:**
- `--project, -p` - Project ID to import users into (required)
- `--from-csv` - Path to CSV file with columns: email, role_id

**Example:**

```bash
$ raps admin user import --project "proj-123" --from-csv users.csv
```

## Folder Commands

### raps admin folder set-permissions

Set folder permissions for a user across multiple projects.

```bash
raps admin folder set-permissions [OPTIONS] <EMAIL>
```

**Arguments:**
- `EMAIL` - User email address

**Options:**
- `-a, --account <ACCOUNT>` - Account ID
- `--permission <LEVEL>` - Permission level (required)
  - `view`, `view_download`, `upload`, `edit`, `control`
- `--folder <TYPE>` - Folder type (required)
  - `project_files`, `plans`, or folder URN for custom
- `-f, --filter <FILTER>` - Filter expression
- `--dry-run` - Preview changes without executing
- `--output <FORMAT>` - Output format

**Example:**

```bash
$ raps admin folder set-permissions "contractor@partner.com" \
    --account "acc-123" --permission edit --folder project_files
```

### raps admin project create

Create a new project in an account.

```bash
raps admin project create [OPTIONS]
```

**Options:**
- `--account, -a` - Account ID
- `--name, -n` - Project name (required)
- `--type, -t` - Project type
- `--classification` - Project classification (production, template, component, sample)
- `--start-date` - Start date (ISO 8601)
- `--end-date` - End date (ISO 8601)
- `--timezone` - Time zone (e.g., "America/New_York")

**Example:**

```bash
$ raps admin project create --account "acc-123" --name "New Hospital Wing" --classification production
```

### raps admin project create-batch

Create multiple projects in parallel with sequential naming.

```bash
raps admin project create-batch [OPTIONS] --prefix <PREFIX> --count <COUNT>
```

**Options:**
- `--account, -a` - Account ID (defaults to `APS_ACCOUNT_ID` env var)
- `--prefix, -p` - Name prefix for projects (e.g., "Test-Project" produces "Test-Project-001", "Test-Project-002", ...)
- `--count, -c` - Number of projects to create
- `--start` - Starting number (default: 1)
- `--concurrency` - Max concurrent requests (default: 20)
- `--no-wait` - Don't wait for project activation (fire-and-forget)

**Example:**

```bash
# Create 50 test projects with concurrency 20
$ raps admin project create-batch --account "acc-123" --prefix "Phase2-Building" --count 50

# Start numbering from 100, don't wait for activation
$ raps admin project create-batch --prefix "Wing" --count 10 --start 100 --no-wait
```

## Common Workflows

### Onboard New Team Member

```bash
# 1. Add to all active projects (simplest approach)
raps admin user add-to-all-projects "new@company.com" \
  --account "$ACCOUNT_ID" --dry-run  # preview first

raps admin user add-to-all-projects "new@company.com" \
  --account "$ACCOUNT_ID"

# 2. Or add to filtered projects with a specific role
raps admin user add "new@company.com" \
  --account "$ACCOUNT_ID" \
  --role "Project Admin" \
  --filter "name:*Active*"

# 3. Grant folder permissions
raps admin folder set-permissions "new@company.com" \
  --account "$ACCOUNT_ID" --permission edit --folder project_files
```

### Offboard Departing Employee

```bash
# 1. Remove from all projects
raps admin user remove "departed@company.com" --account "$ACCOUNT_ID"

# 2. Verify removal
raps admin operation status
```

### Bulk Role Update

```bash
# Update role to "Document Manager" on active 2024 projects
raps admin user update "user@company.com" \
  --account "$ACCOUNT_ID" \
  --role "Document Manager" \
  --filter "name:^2024-"
```

| Code | Description |
|------|-------------|
| `0` | All operations succeeded |
| `1` | Partial success (some items failed) |
| `2` | Operation failed to start |
| `3` | User cancelled operation |
