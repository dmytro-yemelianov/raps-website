## Why the Gap Exists

<div class="not-prose my-8 space-y-4">
  <div class="bg-gradient-to-r from-slate-900/80 to-slate-800/80 rounded-xl p-6 border-l-4 border-blue-500">
    <h4 class="text-white font-bold mb-2">The store was designed for end users</h4>
    <p class="text-slate-400 text-sm">The Autodesk App Store was built for architects, engineers, and construction managers. The submission process, categories, and discovery all optimize for desktop plugin distribution. A CLI tool does not fit this model.</p>
  </div>
  <div class="bg-gradient-to-r from-slate-900/80 to-slate-800/80 rounded-xl p-6 border-l-4 border-purple-500">
    <h4 class="text-white font-bold mb-2">No category for developer tools</h4>
    <p class="text-slate-400 text-sm">The taxonomy itself excludes developer infrastructure. Categories include "Architecture," "Structure," "MEP," "Construction" -- but nothing for "Developer Tools," "API Utilities," or "Automation." If there is no shelf, there is no product.</p>
  </div>
  <div class="bg-gradient-to-r from-slate-900/80 to-slate-800/80 rounded-xl p-6 border-l-4 border-amber-500">
    <h4 class="text-white font-bold mb-2">Separate developer ecosystem</h4>
    <p class="text-slate-400 text-sm">Autodesk's developer community (ADN) is entirely separate from the App Store ecosystem. ADN developers share code on GitHub and Forge blogs. The App Store serves a different audience. The two worlds do not overlap.</p>
  </div>
  <div class="bg-gradient-to-r from-slate-900/80 to-slate-800/80 rounded-xl p-6 border-l-4 border-red-500">
    <h4 class="text-white font-bold mb-2">Result: a structural blind spot</h4>
    <p class="text-slate-400 text-sm">4,078 apps serving one audience. Zero apps serving the people who build integrations, manage infrastructure, and automate workflows against APS APIs. Not a gap in execution -- a gap in the platform's design.</p>
  </div>
</div>


## Methodology

**Scope:** All apps listed on apps.autodesk.com as of March 2026.

**Search terms:** CLI, command-line, terminal, authentication, OAuth, token, MCP, Model Context Protocol, developer tools, API helper, translation debug, bulk admin, scripting, automation SDK.

**Classification:** Each app was checked for (a) command-line interface, (b) scriptability, (c) developer-facing functionality. Apps were classified as "developer tools" only if they met at least one of these criteria.

**Limitations:** This audit covers the public App Store only. Internal Autodesk tools and private enterprise solutions are not included.
